What is a Virtual Machine?

A virtual machine (VM) is practically identical to a physical computer: both strictly have a CPU, RAM, hard drive, internet connection if needed, and can run various operating systems and software. The only difference is that a physical computer uses its own hardware, while a virtual machine borrows resources from its host. Essentially, a VM is code—a virtual computer inside a physical server.

Software called a "hypervisor" is responsible for creating and running virtual machines, as well as distributing resources—CPU, RAM, and disk space—among them.

Advantages & Disadvantages

Advantages

  • Run a different OS without partitioning the hard drive or rebooting.
  • Install legacy OSs like DOS even if physical hardware is incompatible.
  • Transport an entire OS on a USB stick or between computers easily.
  • Isolate doubtful programs or viruses for study without risking the physical machine. VMs can be frozen, reset, copied, or deleted easily. snapshots allow restoring to a previous state.
  • Install multiple VMs on limited physical hardware, saving money, electricity, and maintenance time.

Disadvantages

  • The host computer needs sufficient resources (especially RAM) to support virtualization; otherwise, performance suffers.
  • If the host fails, VMs may become inaccessible without backups on another machine.
  • A physical machine running multiple VMs can be a single point of failure and potentially more vulnerable if not managed correctly.

Debian vs CentOS

For the Born2beroot project, we choose between two GNU/Linux distributions: Debian or CentOS. Both are community-driven, open-source, and renowned for stability and security.

Debian

Started in 1993, Debian is one of the oldest and most robust Linux distributions. It supports a wide variety of architectures and offers a huge selection of packages. It is minimal, resource-efficient, and secure managed by `apt`.

CentOS

CentOS (Community Enterprise Operating System) was the free, open-source version of RHEL (Red Hat Enterprise Linux). It is extremely stable and widely used in enterprise environments. However, recent changes (shift to CentOS Stream) have made it less of a 1:1 clone of stable RHEL.

Choice: This guide focuses on Debian as it is highly prevalent on internet servers and often preferred for stability and ease of package management for this project.

Creating the Virtual Machine

We need Oracle VirtualBox and the Debian ISO.

  • VirtualBox: Official Site
  • Debian ISO (amd64-netinst.iso is sufficient): Official Site

Steps:

  1. Open VirtualBox and click New.
  2. Name: Choose a name (e.g., your_login_42).
  3. Folder: On 42 computers, use /sgoinfre/goinfre/Perso/your_login to save space.
  4. RAM: 1024 MB (default) is usually sufficient.
  5. Hard Disk: Create a virtual hard disk (VDI), Dynamically Allocated.
  6. Size: 12GB is enough for mandatory, but use 30GB if you plan to do bonuses.

Your VM is created!

Configuration & Rules

Sudo Policy

Sudo ("superuser do") allows authorized users to execute commands as the superuser.

dpkg -l | grep sudo # Check installation /var/log/sudo/ # Log directory (check sudo.log)

Password Policy

Enforce strong password rules (expiration, complexity). Check a user's status:

sudo chage -l <username>
  • PASS_MAX_DAYS 30: Password expires every 30 days.
  • PASS_MIN_DAYS 2: Minimum days before changing password.
  • PASS_WARN_AGE 7: Warning 7 days before expiration.

UFW (Uncomplicated Firewall)

Firewall to secure the network. Only port 4242 should be open.

sudo ufw status numbered # List active rules sudo ufw allow 4242 # Open port 4242 sudo ufw enable # Start firewall

SSH (Secure Shell)

Secure remote connection protocol. Should run on port 4242, not 22.

sudo systemctl status ssh # Check status /etc/ssh/sshd_config # Config file (change Port 22 to 4242)

Crontab (Monitoring)

Cron schedules tasks. You need a monitoring script running every 10 minutes.

sudo crontab -u root -e # Edit root's crontab

Add this line to run exactly every 10 minutes:

*/10 * * * * /path/to/monitoring.sh

Defense Preparation

Theoretical Questions

  • How does a VM work? It's a virtual computer using resources (CPU, RAM) borrowed from the physical host via a Hypervisor.
  • Apt vs Aptitude? Both are package managers. Aptitude is smarter: it cleans up unused dependencies automatically. Apt is lower-level and only does exactly what's requested.
  • What is AppArmor? mandatory Access Control (MAC) system. It restricts programs' capabilities (like file access) to increase security. Included by default in Debian.
  • LVM (Logical Volume Manager)? Allows flexible resizing of partitions across multiple physical disks without formatting.
